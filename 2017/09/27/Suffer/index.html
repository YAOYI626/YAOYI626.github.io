终于成功以及经验总结

发表于 2017-09-27 | 分类于 菜鸡的逆袭记录 | 本文总阅读量 次

为了搭这个博客小白的我真的费了一番功夫，下面回忆整理一下我遇到的困难并给出解决方案，希望其他小白看到后能不像我一样花那么多时间。

要点版本

  1. 使用稳定的网络，不要轻易使用镜像源，会给后期操作带来一定的不便。

  2. Hexo配置_config.yml时不要忘记冒号后的空格。

    举个我的_config.yml的例子：

    deploy: 

   type: git

   repository: git@github.com:YAOYI626/YAOYI626.github.io.git

   branch: master

**每个冒号后面都要有空格**

  1. 某些教程已经比较落后，在Hexo3.0及以上版本中仓库地址不能使用http，而必须用git@github.com作为开头:

  2. github建仓库时必须与用户名相同，否则无法关联到该Github Pages。

  3. 使用主题时尽量下载成熟版本的主题。否则可能会因为主题的不完善导致网页乱码。

下面是吐槽

  1. 最大也是最先遇到的困难是还没有带自己的电脑，于是饱受网吧及机房机器和网络的困扰。在第一天我在网吧尝试安装hexo时，可能是因为网络的原因，hexo竟然安装不下来，卡死在第一次换行后。在这种情况下我使用了在网上找到的淘宝镜像源安装hexo，然而下来的东西却是残次品，配置完成后竟然找不到config文件，浪费了一下午的时间。

  2. 今天下午总算在机房安装了配套环境，开始搭建。然后就是hexo d出现错误，看了半天发现是config文件冒号后忽略了空格。

  3. 但是hexo g和d之后我的github网站竟然点不开了，看了半天原来是仓库名和用户名没有匹配，修改之后成功打开了远程网站。

  4. 在hexo官网上挑选了主题，可能是因为主题没有开发成熟的原因导致部署失败，网页出现乱码，幸好提前保存了副本，恢复原件后选用了比较成熟的NEXT作为主题并小小的做了个性化配置，效果还算不错。

  5. 搞完了这些以后我还是很有成就感的（大佬们不要吐槽了…）。
